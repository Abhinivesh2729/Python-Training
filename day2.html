Python's History and Evolution

Python was created by Guido van Rossum in the late 1980s and was first released in 1991. The name "Python" was inspired by the British comedy group Monty Python, not the snake!

1989

Guido van Rossum begins working on Python as a hobby project during Christmas holidays.

1991

Python 0.9.0 is released, the first public release.

2000

Python 2.0 is released with new features like list comprehensions and garbage collection.

2008

Python 3.0 is released with a focus on removing duplicate features and improving consistency.

2019

Python 2 reaches end of life, with Python 3 becoming the standard.

2020-Present

Python becomes one of the most popular programming languages with wide adoption in data science, AI, web development and automation.

Python Philosophy

Python's design philosophy emphasizes code readability with its notable use of significant whitespace. Its core principles are summarized in "The Zen of Python" by Tim Peters, which can be viewed by typing import this in Python.

Key Features of Python:

  • Simplicity: Easy-to-read syntax that resembles English
  • Versatility: Used in web development, data analysis, AI, automation, and more
  • Interpreted: No compilation needed, making development faster
  • Dynamically Typed: No need to declare variable types
  • Large Standard Library: "Batteries included" philosophy with many built-in modules
  • Strong Community: Extensive documentation and third-party packages

How to Run Python: Installation Guide

Before we can start writing Python code, we need to install Python on our computer. Python is freely available for all major operating systems (Windows, macOS, Linux).

System Requirements:

  • Windows 7 or later
  • macOS 10.11 or later
  • Almost any modern Linux distribution
  • At least 100 MB of disk space
  • Download Python: Visit python.org/downloads and download the latest version for your operating system (Python 3.11+ recommended).
  • Run the Installer: Double-click the downloaded file to start the installation process.
  • Important Options: On Windows, check "Add Python to PATH" before clicking "Install Now" to ensure Python can be run from the command line.
  • Verify Installation: Open a command prompt (Windows) or terminal (macOS/Linux) and type python --version or python3 --version to confirm Python was installed successfully.

    • Important Note for Windows Users

    Make sure to check the "Add Python to PATH" option during installation! This allows you to run Python from any Command Prompt window.

    Understanding the Python Interpreter

    How Python Code Works in a Computer

    Understanding how Python works "under the hood" is essential for every Python developer. Let's explore the workflow from writing code to execution:

    Step 1: Writing Code

    You write Python code in a text file with a .py extension using any text editor or IDE (Integrated Development Environment).

    Step 2: Python Interpreter

    The Python interpreter reads your code and converts it into an intermediate language called "bytecode" (files with .pyc extension).

    Step 3: Python Virtual Machine

    The Python Virtual Machine (PVM) executes the bytecode instruction by instruction, converting it into machine code that your computer understands.

    Step 4: Output

    The results of your code are displayed as output on the screen or saved to files, depending on your program's purpose.

    What is an Interpreter?

    An interpreter is a program that directly executes code written in a programming language without requiring it to be compiled into machine language first. The Python interpreter translates and runs your code line by line.

    For Non-Programmers

    Think of the interpreter as a translator who reads your instructions (code) in Python and translates them into a language the computer understands, executing each instruction immediately before moving to the next one.

    Interpreter vs. Compiler: Key Differences

    Feature Interpreter (Python) Compiler (C, Java)
    Execution Line-by-line execution Converts entire program to machine code before execution
    Speed Generally slower execution Faster execution
    Development Faster development cycle More time-consuming development cycle
    Error Detection Stops at first error Shows all errors at once after compilation
    Memory Usage Often uses more memory Usually more memory-efficient
    Portability Highly portable ("Write once, run anywhere") May need recompilation for different platforms

    IDLE: Python's Integrated Development Environment

    What is IDLE?

    IDLE (Integrated Development and Learning Environment) is Python's built-in, basic IDE that comes pre-installed with Python. It's perfect for beginners and provides all the essential features needed to start programming in Python.

    Code Editor

    Write and edit Python code with syntax highlighting, making code easier to read and understand.

    Interactive Shell

    Test code snippets and experiment with Python commands in real-time, with immediate feedback.

    Debugger

    Find and fix errors in your code with built-in debugging tools.

    IDLE Installation Guide

    The good news is that if you've already installed Python using the official installer, IDLE is already installed on your computer! Here's how to access it:

  • On Windows: Click Start menu → search for "IDLE" or navigate to Python in your programs list.
  • On macOS: Open Applications folder → Python folder → IDLE icon, or use Spotlight (Cmd+Space) to search for "IDLE".
  • On Linux: Open terminal and type idle or idle3, or look for IDLE in your applications menu.

    • Troubleshooting

    If IDLE doesn't launch, make sure Python is properly installed. You might need to reinstall Python or check if your PATH environment variable is correctly set.

    IDLE First Look

    When you open IDLE, you'll see two main windows:

    Python Shell Window

    This is where you can type Python commands and see immediate results. It's also called "Interactive Mode." The shell displays the Python version and shows a prompt (>>>) where you can type commands.

    Editor Window

    This opens when you create a new file or open an existing one. Here you can write and save complete Python programs. To open it, click File → New File in the Shell window.

    Pro Tip

    In IDLE, you can run your code by pressing F5 or by clicking Run → Run Module. This automatically saves your file before execution.

    Online Python Compilers: Code Without Installation

    Online Python compilers (or interpreters) are web-based platforms that let you write and run Python code directly in your browser. They're perfect for beginners or situations where you can't install Python locally.

    Benefits of Using Online Compilers

    • No Installation Required: Start coding immediately without installing anything on your computer
    • Accessibility: Access your code from any device with an internet connection
    • Sharing: Easily share your code with others via unique URLs
    • Collaboration: Some platforms allow real-time collaboration
    • Learning: Perfect for following along during class without setup hassles

    Limitations of Online Compilers

    While convenient, online compilers have some restrictions:

    • Limited access to certain Python libraries
    • Execution time/memory constraints
    • Potential connectivity issues
    • Less suitable for large-scale projects

    For serious development, a local Python installation is recommended.

    Interactive vs. Script Mode: Two Ways to Use Python

    Python can be used in two primary modes: Interactive Mode and Script Mode. Each has its own purpose and advantages.

    Interactive Mode

    Also known as the Python Shell or REPL (Read-Evaluate-Print Loop), this mode allows you to type commands one at a time and see immediate results.

    How to Access:

    • Type python or python3 in your command line/terminal
    • Open IDLE, which starts in interactive mode by default
    • Use the interactive console in online compilers

    Best Used For:

    • Testing quick ideas or short code snippets
    • Exploring Python features
    • Debugging and testing individual functions
    • Learning and experimenting with Python
    Python Interactive Mode Example 
    >>> print("Hello, Python!")
Hello, Python!
>>> 2 + 3
5
>>> name = "Student"
>>> print(f"Welcome, {name}!")
Welcome, Student!

    Script Mode

    In Script Mode, you write your entire program in a file with a .py extension and execute it all at once.

    How to Use:

    • Create a file with .py extension using any text editor or IDE
    • Write your complete Python program
    • Execute by running python filename.py in terminal or using the Run button in IDEs

    Best Used For:

    • Writing complete programs
    • Creating reusable code
    • Developing larger applications
    • Saving work for future use
    Python Script Example (hello.py) 
    # This is a simple Python script
print("Hello, Python!")

# Calculating and storing values
result = 2 + 3
print(f"2 + 3 = {result}")

# Using variables
name = "Student"
print(f"Welcome, {name}!")

    Python File Saving and Loading

  • Creating a New File in IDLE:

    Open IDLE → Click File → New File → Write your code → Save with File → Save As → Give a name with .py extension

  • Running Your Python File:

    With the file open in IDLE editor, press F5 or click Run → Run Module

  • Loading an Existing File:

    In IDLE, click File → Open → Navigate to your Python file and select it

  • Running from Command Line/Terminal:

    Navigate to the directory containing your file → Type python filename.py or python3 filename.py

    • Good Practice

    Organize your Python files in dedicated folders based on projects or topics. This makes it easier to manage your code as you learn more and create more complex programs.

    Hello, World!: Your First Python Program

    Creating a "Hello, World!" program is a programming tradition when learning a new language. It's a simple program that outputs the text "Hello, World!" to the screen.

    Why "Hello, World!"?

    This simple program serves multiple purposes:

    • Verifies that your Python environment is set up correctly
    • Introduces the basic syntax for outputting text
    • Provides a quick win to build confidence
    • Follows a programming tradition dating back to the 1970s

    Creating Your First Python Program

    In Interactive Mode

    Python Interactive Shell 
    >>> print("Hello, World!")
Hello, World!

    Simply type the command at the Python prompt (>>>) and press Enter. The output appears immediately below.

    In Script Mode

    hello_world.py 
    # My first Python program
print("Hello, World!")

# Adding a personal touch
print("Welcome to Python programming!")

    Save this code in a file named hello_world.py and run it using IDLE or the command line.

    Understanding the Code

    • print() is a built-in Python function that displays output on the screen
    • "Hello, World!" is a string (text) enclosed in quotation marks
    • Lines starting with # are comments; Python ignores them when running

    Fun Fact

    In Python 2 (now obsolete), the print statement didn't require parentheses: print "Hello, World!". In Python 3, print is a function and requires parentheses: print("Hello, World!"). This is one of many reasons why Python 3 is more consistent.

    Expanding Your "Hello, World!"

    Let's try a slightly more advanced version that uses variables:

    enhanced_hello.py 
    # Using variables in our Hello World program
name = "Python Learner"
day = 2
course = "Python Training"

# Using formatted string to combine text and variables
print(f"Hello, {name}!")
print(f"Welcome to Day {day} of your {course}!")
print("You've successfully run your first Python program!")

    Practice Assignments

    Try these exercises to reinforce what you've learned:

    1. Create a hello_world.py file and run it successfully
    2. Modify it to display your name
    3. Add more print statements with different messages
    4. Try using variables to store your name and age, then display them
    5. Experiment with both interactive mode and script mode

    Common Beginner Mistakes

    • Forgetting the parentheses in print()
    • Mismatched quotation marks
    • Forgetting to save the file before running it
    • Using the wrong file extension (use .py)
