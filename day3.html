Hello World in Python

The traditional first program in any programming language is "Hello, World!". This simple program displays the message "Hello, World!" to the user. In Python, this is remarkably simple to do:

hello_world.py 
print("Hello, World!")
Output:
Hello, World!

This single line of code uses the built-in print() function to display text in the console. The text inside the quotation marks is a string - a sequence of characters.

Understanding the Code

When you run this program, Python's interpreter executes the statement from top to bottom. It sees the print() function call and evaluates what's inside the parentheses (the "Hello, World!" string), then displays that output to the console.

Running Your First Python Program

There are two main ways to run Python code:

  1. Interactive Mode: Typing commands directly into the Python interpreter.
    Python Interactive Shell 
    >> print("Hello, World!")
Hello, World!
>>>
  2. Script Mode: Writing code in a .py file and then executing it.
    Terminal/Command Prompt 
    $ python hello_world.py
Hello, World!

Pro Tip

For experienced developers, Python's interactive mode is extremely useful for testing small code snippets or ideas without creating a full program file. It's like having a scratchpad for code!

The print() Function in Detail

The print() function is one of the most frequently used functions in Python, especially when you're starting out. It allows you to display information to the user and is invaluable for debugging.

Basic Print Statements

print_examples.py 
# Printing a string
print("Hello, Python!")

# Printing multiple items
print("Python", "is", "awesome")

# Printing numbers
print(42)
print(3.14159)

# Printing variables
name = "Alice"
age = 30
print(name)
print(age)
print(name, "is", age, "years old")
Output:
Hello, Python! Python is awesome 42 3.14159 Alice 30 Alice is 30 years old

Formatting Output with print()

The print function has several parameters that allow you to control how output is formatted:

print_formatting.py 
# Changing the separator between items
print("Python", "is", "awesome", sep="-")

# Changing the end character (default is newline)
print("Hello", end=" ")
print("World!")

# Combining both
print("One", "Two", "Three", sep=" | ", end=" >>> ")
print("Next line")

# Using f-strings (Python 3.6+) for more complex formatting
name = "Bob"
age = 25
print(f"{name} is {age} years old and will be {age + 10} in ten years.")
Output:
Python-is-awesome Hello World! One | Two | Three >>> Next line Bob is 25 years old and will be 35 in ten years.

Key Points About print()

The print() function is highly versatile. The sep parameter defines what separates multiple items (default is a space), and the end parameter defines what is printed at the end (default is a newline).

F-strings (formatted string literals) introduced in Python 3.6 are a powerful way to embed expressions inside string literals. They are prefixed with an 'f' and use curly braces to contain expressions that will be replaced with their values.

Pro Tip

F-strings are not only more readable than older formatting methods but also more efficient. They allow for direct expression evaluation inside the string format, making your code cleaner and easier to maintain.

Comments in Python

Comments are lines of text in your code that are ignored by the Python interpreter. They serve as notes or explanations about the code's purpose and functionality. Good commenting practices make your code more maintainable and help others (and your future self) understand your code.

Single-Line Comments

In Python, single-line comments start with the hash character (#):

comments_example.py 
# This is a single-line comment
print("Hello, Python!")  # This is an inline comment

# Comments can be used to explain what the code does
# Multiple single-line comments can be stacked
# like this to form multi-line comments
print("Comments are not executed by the interpreter")

Multi-line Comments / Docstrings

Python doesn't have a specific syntax for multi-line comments. However, triple-quoted strings (''' or """) that aren't assigned to a variable can function as multi-line comments. These are more commonly used as documentation strings (docstrings):

docstrings_example.py 
"""
This is a multi-line comment using triple double-quotes.
It can span multiple lines and is often used for
documentation at the beginning of files, functions, or classes.
"""

def greet(name):
    """
    This is a docstring for the greet function.
    It explains what the function does.
    
    Parameters:
    name (str): The name of the person to greet
    
    Returns:
    str: A greeting message
    """
    return f"Hello, {name}!"

print(greet("Python Learner"))
Output:
Hello, Python Learner!

Best Practices for Comments

  • Write comments that explain why rather than what (the code itself shows what it does)
  • Keep comments up-to-date when you modify code
  • Use docstrings for functions, classes, and modules to describe their purpose and usage
  • Don't overcomment - clean, readable code often needs fewer comments

Pro Tip

Docstrings are not just comments! They're accessible at runtime through an object's __doc__ attribute and are used by documentation tools. Well-written docstrings make your code more professional and usable by others.

Python Basic Syntax

Python's syntax is designed to be clean and readable. Understanding the basic syntax rules is essential for writing correct Python code.

Statements and Expressions

A statement is a complete line of code that performs an action. An expression is a combination of values, variables, and operators that evaluates to a value.

syntax_examples.py 
# This is a statement
x = 10

# This is an expression
y = x + 5

# This line contains both a statement and an expression
print(x * y)  # The statement is 'print(x * y)' and the expression is 'x * y'

Indentation and Whitespace

Unlike many other programming languages that use braces to define blocks of code, Python uses indentation (whitespace at the beginning of a line):

indentation_example.py 
# Correct indentation is crucial in Python
if True:
    print("This is indented correctly")
    print("This is in the same block")
    
    if True:
        print("Another level of indentation")
        
print("Back to the first level")

# Incorrect indentation will cause errors
if True:
    print("This is correct")
  # print("This would cause an IndentationError")  # Uncomment to see error
Output:
This is indented correctly This is in the same block Another level of indentation Back to the first level

Basic Code Organization

Python code is organized in a hierarchical structure:

organization_example.py 
# 1. Import statements come first
import math

# 2. Constants and global variables
PI = 3.14159
MAX_VALUE = 100

# 3. Function and class definitions
def calculate_area(radius):
    """Calculate the area of a circle."""
    return PI * radius * radius

# 4. Main code execution
if __name__ == "__main__":
    radius = 5
    area = calculate_area(radius)
    print(f"The area of a circle with radius {radius} is {area:.2f}")
Output:
The area of a circle with radius 5 is 78.54

Python Syntax Key Points

  • Python uses indentation to define code blocks
  • The standard indentation is 4 spaces (not tabs)
  • Statements usually occupy one line each
  • Line continuation is possible with the backslash (\) character or within parentheses, brackets, or braces
  • Python is case-sensitive: variable and Variable are different

Pro Tip

Most Python developers follow the PEP 8 style guide, which provides conventions for writing clean, readable Python code. Using tools like black or flake8 can help ensure your code follows these conventions.

Example 2: Temperature Converter

temp_converter.py 
# Temperature converter: Fahrenheit to Celsius
# Formula: C = (F - 32) * 5/9

# Function definition with docstring
def fahrenheit_to_celsius(fahrenheit):
    """
    Convert temperature from Fahrenheit to Celsius.
    
    Parameters:
    fahrenheit (float): Temperature in Fahrenheit
    
    Returns:
    float: Temperature in Celsius
    """
    celsius = (fahrenheit - 32) * 5/9
    return celsius

# Main program
if __name__ == "__main__":
    # Get user input
    fahrenheit = float(input("Enter temperature in Fahrenheit: "))
    
    # Convert temperature
    celsius = fahrenheit_to_celsius(fahrenheit)
    
    # Display result
    print(f"{fahrenheit}°F is equal to {celsius:.2f}°C")
Sample Run:
Enter temperature in Fahrenheit: 98.6 98.6°F is equal to 37.00°C
